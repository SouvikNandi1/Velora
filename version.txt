v1.0.0
- Initial working prototype of PyQt6 Powerful Terminal.

v1.1.0
- Added terminal search functionality (Ctrl+F) to easily locate text in the buffer.
- Added a Status Bar to track application state and font size changes dynamically.

v1.1.1
- Improved Context Menu design with proper styling to match the dark theme.
- Fixed Context Menu bugs: "Copy" and "Paste" actions are now contextually enabled/disabled based on text selection and clipboard content respectively.

v1.2.0
- Added a Settings Dialog to customize terminal preferences.
- Implemented persistent settings (QSettings) for Font Family and Font Size.
- Added a Settings option to the right-click context menu.

v1.2.1
- Fixed a bug where the Settings Dialog crashed on launch by updating `QFontDatabase.systemFontFamilies()` to the correct PyQt6 `QFontDatabase.families()` method.

v1.3.0
- Added multi-tab support with `QTabWidget`, enabling multiple simultaneous terminal sessions.
- Implemented a dynamic Theme Engine with presets including Dracula, Solarized Light, and Hacker.
- Refactored core logic into a new `TerminalSession` component for modularity.
- Added global shortcuts `Ctrl+T` to open a new tab and `Ctrl+W` to close the current tab.

v1.3.1
- Fixed an import error causing the app to crash on launch by moving `QShortcut` to the correct `PyQt6.QtGui` module.

v1.3.2
- Fixed a bug where `Ctrl+T` and `Ctrl+W` shortcuts were silently ignored due to Python Garbage Collection.
- Added `Ctrl+Tab` and `Ctrl+Shift+Tab` shortcuts for cycling forward and backward through open tabs.

v1.3.3
- Enforced `ApplicationShortcut` context for all tab navigation shortcuts (`Ctrl+T`, `Ctrl+W`, etc.). This guarantees the shortcuts will trigger globally, preventing child text-edit widgets from accidentally swallowing the keystrokes.

v1.3.4
- Redesigned the Tab Bar UI to feature rounded corners, depth elevation for active tabs, and interactive hover states.

v1.3.5
- Fixed an OS-level issue where pressing `Ctrl+T` suspended the application (`[1]+ Stopped`). Resolved by ignoring `SIGTTOU`/`SIGTTIN` signals and disabling bash job control (`+m`) to prevent background terminal hijacking.

v1.3.6
- Tabs now close cleanly and automatically when the underlying shell session exits (e.g., when typing `exit` or pressing `Ctrl+D`).
- Added a "+" button to the top right corner of the tab bar for easily opening new tabs with the mouse.

v1.3.7

v1.3.8
- Fixed a critical bug where new tabs lacked essential OS environment variables (like `$PATH` and `$HOME`) due to an empty environment spawn. This caused commands to fail ("command not found") in newly created tabs.
- Fixed a PyQt6 `TypeError` when closing tabs by correctly passing `exit_code` and `exit_status` through the `QProcess.finished` lambda.
- Hardened tab navigation signals with `*args` to safely drop unexpected arguments from button clicks and shortcuts.

v1.3.9
- Resolved the persistent "new tab freezing" issue by spawning child shells inside a true Pseudo-Terminal (PTY) using Python's `pty` module.
- This completely isolates each tab's shell from the parent's TTY, guaranteeing interactive prompts and physically preventing all OS-level background suspension (`SIGTTOU`/`SIGTTIN`) conflicts.
